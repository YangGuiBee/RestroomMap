# 화장실SOS(RestroomMap) 버전1 상세 개발계획

> Claude Code로 개발하는 "대한민국 공공화장실 맵" — 모바일 웹앱(PWA)
> 작성일: 2026-07-17

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 제품명 | 화장실SOS (RestroomMap) |
| 버전1 목표 | 현재 위치 기준 가장 가까운 공공화장실을 **3초 안에** 찾고, 원터치로 내비 안내까지 연결 |
| 플랫폼 | 모바일 웹앱(PWA) — 브라우저 즉시 실행, 홈화면 설치 가능, 차량 인포테인먼트 브라우저에서도 동작 |
| 핵심 사용자 | 운전 중 급하게 화장실이 필요한 사람 (본인 포함, IBS 등) |
| 개발 도구 | Claude Code |
| 버전1 비용 | **0원 가능** (공공데이터 무료 + 지도 API 무료쿼터 + 정적 호스팅 무료) |

### 핵심 UX 원칙 (긴급 상황용 앱이므로)
1. **열자마자 답이 보인다** — 앱 실행 즉시 GPS 잡고 가장 가까운 화장실 3곳을 지도+리스트로 표시. 검색어 입력 불필요.
2. **버튼은 크게, 단계는 적게** — 목표: 실행 → 확인 → "안내 시작" 탭, 총 2번의 터치.
3. **운전 중 사용 고려** — 큰 글씨, 고대비, 거리·소요시간 강조. (안전 문구: "정차 후 조작하세요")

## 2. 데이터 소스

### 2.1 공중화장실정보 (localdata.go.kr, 지방행정 인허가 데이터)
- 다운로드 완료: `공중화장실정보.csv` (53,543건, CP949 인코딩, 34컬럼)
- 주요 필드: 화장실명, 소재지 도로명/지번 주소, 구분(공중/개방/이동/간이), 개방시간, 남/녀 변기 수, 장애인용, 기저귀교환대, 비상벨, CCTV, 관리기관, 전화번호
- **⚠ 위도/경도 컬럼 없음.** 표준데이터(data.go.kr/15012892)도 2025년 2월부터 원천 정책 변경으로 WGS84 위경도 제공이 중단됨 → **지오코딩이 필수**

### 2.2 수집 전략: "지오코딩 + 정적 스냅샷" 방식
화장실 위치는 실시간으로 변하지 않으므로, 좌표 변환을 1회 배치로 수행하고 정적 파일로 배포.

```
[공중화장실정보.csv] → (카카오 로컬 API 지오코딩: 주소→위경도) → (정제 스크립트) → [static JSON 타일] → 앱에 번들/CDN 배포
```

- 지오코딩: 카카오 로컬 API(REST 키), 무료 쿼터 일 10만 건 — 53,543건 1일 내 처리 가능
- 도로명주소 실패 시 지번주소로 재시도, 둘 다 실패 건은 별도 리포트

- 장점: API 키 노출 걱정 없음, 오프라인 동작 가능, 응답속도 최고, 쿼터 소진 없음
- 갱신: 월 1회 스크립트 재실행 (추후 GitHub Actions로 자동화)

### 2.3 데이터 정제 규칙 (스크립트에 반영)
- CP949 → UTF-8 변환
- 지오코딩 실패 행 별도 관리(수동 보정 대상), 좌표 범위 검증(한국 영역: 위도 33~39, 경도 124~132)
- 중복(동일 좌표+유사 명칭) 병합
- 개방시간 문자열 정규화 ("24시간", "09:00~18:00" 등 → 파싱 가능한 형태)
- 파일 분할: 전국 1개 JSON(수 MB)은 무거우므로 **geohash 격자 또는 시도 단위로 분할**하여 현재 위치 주변 타일만 로드

## 3. 기술 스택

| 레이어 | 선택 | 이유 |
|---|---|---|
| 프론트엔드 | **React + Vite + TypeScript** | Claude Code가 가장 잘 다루는 조합, PWA 플러그인(vite-plugin-pwa) 지원 |
| 지도 | **카카오맵 JavaScript API** ([가이드](https://apis.map.kakao.com/web/guide/)) | 국내 도로·POI 정확도, 무료 쿼터, 카카오내비 딥링크와 자연 연계. ※ 2026-07-21부터 쿼터 정책 변경 — 앱 등록 시 비즈월렛 연결 필요 여부 확인 |
| 지도 대안 | Leaflet + OpenStreetMap | 완전 무료·무키. 버전4~5(해외) 대비 겸용. 국내 상세도는 카카오보다 낮음 |
| 데이터 | 정적 JSON (격자 분할) | 2.2 참조. 백엔드 서버 불필요 |
| 백엔드 | **없음 (버전1)** | 정적 호스팅만으로 완성. 버전2(신청 접수)부터 도입 |
| 호스팅 | Vercel 또는 Cloudflare Pages (무료) | HTTPS 기본 제공 — **Geolocation API는 HTTPS 필수** |
| PWA | vite-plugin-pwa (Workbox) | 홈화면 설치, 데이터 타일 오프라인 캐시 |

## 5. 기능 명세 (버전1)

### 필수 (MVP)
| ID | 기능 | 상세 |
|---|---|---|
| F1 | 현재 위치 탐지 | Geolocation API. 거부/실패 시 지도 중심 기준으로 대체 + 주소 검색 폴백 |
| F2 | 주변 화장실 표시 | 반경 자동 확장(500m→1km→3km→5km) 검색, 지도 마커 + 거리순 리스트 |
| F3 | 최단 3곳 하이라이트 | 상단에 "가장 가까움" 카드 크게 표시 (명칭·직선거리·개방시간) |
| F4 | 상세 정보 | 마커/카드 탭 → 개방시간, 남녀 변기 수, 장애인용, 기저귀교환대, 관리기관 전화 |
| F5 | **원터치 길안내** | 카카오내비/티맵/카카오맵 딥링크(`kakaonavi://`, `tmap://`, URL scheme). 미설치 시 카카오맵 웹 길찾기로 폴백 |
| F6 | PWA 설치·오프라인 | 마지막 조회 지역 타일 캐시 → 터널·음영지역에서도 리스트 확인 가능 |

### 선택 (MVP 이후, 버전1.x)
- F7 즐겨찾기(localStorage 아님 — PWA는 가능하나 IndexedDB 권장)
- F8 "지금 열려있는 곳만" 필터 (개방시간 파싱 기반)
- F9 다크모드(야간 운전)
- F10 음성 안내("가장 가까운 화장실 350m")

**① 메인(지도) 화면** — 상단: 현재 주소 + [내 위치] 버튼 / 중앙: 지도(마커) / 하단: 가까운 순 카드 3장(스와이프), 각 카드에 큰 [안내 시작] 버튼

## 7. Claude Code 단계별 개발 계획

각 Phase는 Claude Code 세션 1~2회 분량. **Phase마다 동작 확인 후 다음 진행** (git commit 단위).

### Phase 0 — 프로젝트 셋업 (0.5일)
- 리포지토리 생성, Vite+React+TS 스캐폴딩, CLAUDE.md 작성(아래 8장)
- 프롬프트 예시:
```
Vite + React + TypeScript로 PWA 프로젝트를 초기화해줘.
vite-plugin-pwa를 설정하고, 모바일 우선 레이아웃 기본 골격을 만들어줘.
```

### Phase 1 — 데이터 파이프라인 (1일)
- 공공데이터포털에서 CSV 수동 다운로드 → `data/raw/`에 배치
- 프롬프트 예시:
```
data/raw/toilet.csv (전국공중화장실표준데이터)를 읽어서
1) 위경도 결측·범위이탈 행 제거, 2) 중복 병합, 3) 개방시간 정규화 후
geohash 5자리 단위로 분할된 public/tiles/{geohash}.json 파일들을 생성하는
Node 스크립트 scripts/build-tiles.ts를 만들어줘. 정제 리포트(제거 건수)도 출력해줘.
```
- 산출물 검증: 총 건수, 시도별 분포, 좌표 샘플 지도 확인

### Phase 2 — 지도 + 현재 위치 (1일)
- 카카오 developers 앱 등록, JavaScript 키 발급, 도메인 등록
- 프롬프트 예시:
```
카카오맵 JS SDK를 붙여서 전체 화면 지도를 띄우고,
Geolocation으로 현재 위치를 받아 지도 중심 이동 + 내 위치 마커를 표시해줘.
위치 권한 거부 시 서울시청 기준으로 폴백하고 안내 배너를 띄워줘.
```

### Phase 3 — 주변 검색 + 리스트 (1~2일)
```
현재 위치의 geohash 주변 타일들을 fetch해서 Haversine 거리 계산으로
가까운 순 정렬 후, 지도 마커와 하단 카드 리스트(상위 3곳 강조)를 구현해줘.
반경 내 결과가 없으면 500m→1km→3km→5km로 자동 확장해줘.
```

### Phase 4 — 상세 + 내비 딥링크 (1일)
```
카드 탭 시 바텀시트로 상세정보를 보여주고, [안내 시작] 버튼에
카카오내비 딥링크 → 실패 시 티맵 → 최종 폴백 카카오맵 웹 길찾기 순서로 연결해줘.
[전화] 버튼은 tel: 링크로.
```

### Phase 5 — PWA·오프라인·마감 (1일)
```
Service Worker로 조회한 타일 JSON을 캐시하고, 오프라인일 때
마지막 데이터로 리스트가 보이게 해줘. 매니페스트(아이콘, 이름 '화장실맵') 완성,
다크모드, "정차 후 조작" 안내 문구 추가.
```

### Phase 6 — 배포 + 실차 테스트 (0.5일)
- Vercel 배포, 카카오 개발자 콘솔에 배포 도메인 등록
- 실제 운전 경로에서 GPS 정확도·딥링크 동작 테스트

**총 예상: 5~7일 (퇴근 후 작업 기준 2~3주)**

## 8. CLAUDE.md 초안 (리포지토리 루트에 배치)

```markdown
# 화장실맵 (RestroomMap)

긴급 상황에서 가장 가까운 공공화장실을 찾는 모바일 PWA.

## 스택
- Vite + React + TypeScript + vite-plugin-pwa
- 지도: 카카오맵 JS SDK (키는 .env, 커밋 금지)
- 데이터: public/tiles/{geohash}.json 정적 파일 (scripts/build-tiles.ts로 생성)

## 원칙
- 긴급용 앱: 실행→안내시작까지 터치 2번 이내 유지
- 모바일 우선, 터치 타깃 최소 48px, 고대비
- 백엔드 없음. 서버 필요한 기능 제안 금지 (버전2에서 도입 예정)
- 위치 데이터는 기기 밖으로 전송하지 않음 (프라이버시)

## 명령어
- npm run dev / npm run build
- npm run tiles  # CSV → 타일 JSON 재생성
```

## 11. 버전2~5 연계 포인트 (버전1에서 미리 심어둘 것)

- **버전2(추가 신청)**: "이 근처 화장실 제보" 버튼 자리만 UI에 확보. 데이터 스키마에 `source: public|user` 필드 미리 포함
- **버전2(관리자 대시보드)**: 로그인 필요한 관리자 화면 + 공공데이터포털 CSV 재다운로드 실행 + 맵 사용로그(조회/안내시작 집계) 현황 확인. **백엔드·인증 시스템이 필요해 V1 "백엔드 없음" 원칙과 충돌 → V2에서 정식 구현.** 위치정보(GPS 좌표) 자체는 절대 로그로 전송하지 않는다는 프라이버시 원칙 유지, 화장실 ID/이름 단위의 익명 집계만 허용. (참고: V1 임시 대안으로 GitHub Actions 수동 트리거 + Vercel Web Analytics 조합을 검토했으나 이것도 정식 관리자 화면은 아니므로 보류)
- **버전3(지하철)**: 타일 스키마에 `category` 필드(공공/지하철/민간) 예약
- **버전4~5(다국어·해외)**: 문자열을 처음부터 i18n 구조(ko.json)로 작성, 지도 레이어 추상화(카카오↔OSM 교체 가능하게)
