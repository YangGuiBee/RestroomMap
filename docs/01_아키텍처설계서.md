## 1. 시스템 개요

운전 중 급하게 화장실이 필요한 사용자가 현재 위치 기준 가장 가까운 공공화장실을 찾아 원터치로 내비게이션 안내까지 연결하는 모바일 우선 PWA(Progressive Web App)이다. **백엔드 서버 없이** 정적 호스팅만으로 동작하며, 화장실 데이터는 빌드 타임에 미리 지오코딩·가공해 정적 JSON 파일로 배포한다.

### 1.1 핵심 설계 원칙
- **백엔드 없음(버전1)**: 서버 비용·운영 부담 없이 정적 호스팅(Vercel)만으로 서비스
- **긴급용 UX**: 실행 → 확인 → "안내 시작"까지 터치 2회 이내
- **위치정보 미전송**: 사용자의 GPS 좌표를 외부 서버로 전송하지 않음(카카오맵 SDK가 지도 표시를 위해 처리하는 것 외 별도 수집·로깅 없음)
- **레이어 추상화**: 지도 SDK(현재 카카오맵), 데이터 소스(현재 정적 JSON)를 교체 가능하도록 인터페이스 분리
- **정보 은닉 금지**: 파싱 실패·불확실한 데이터도 숨기지 않고 원문 그대로 노출(예: 개방시간 원문, "정보없음" 명시)

## 2. 기술 스택

| 레이어 | 기술 | 비고 |
|---|---|---|
| 프론트엔드 프레임워크 | React 18 + TypeScript | Vite 빌드 |
| 빌드 도구 | Vite 5 | 개발 서버(HMR) + 프로덕션 번들링 |
| PWA | vite-plugin-pwa (Workbox) | 서비스워커 자동 생성, 오프라인 캐시 |
| 지도 SDK | 카카오맵 JavaScript SDK | services 라이브러리(장소검색) 포함 |
| 상태관리 | React 내장 useState/useEffect/useMemo | 별도 상태관리 라이브러리 없음(KISS) |
| 데이터 형식 | 정적 JSON (geohash 타일 분할) | `public/tiles/{geohash5}.json` |
| 지오코딩(빌드타임) | 카카오 로컬 API(REST) → VWorld API(폴백) | Node.js 스크립트 |
| 호스팅/배포 | Vercel (Hobby 플랜) | GitHub 연동 자동배포(push 시 재배포) |
| 소스 관리 | GitHub | `YangGuiBee/RestroomMap`, main 브랜치 |
| 언어 | 한국어 단일(i18n 구조 사전 적용) | `src/i18n/ko.json` |

## 5. 데이터 파이프라인

```
공중화장실정보.csv (원본, CP949)
   │  (인코딩 변환)
   ▼
data/공중화장실정보_utf8.csv
   │  scripts/geocode.mjs (카카오 REST → VWorld 폴백, resume 지원)
   ▼
data/geocoded.jsonl (52,274건, 97.6% 성공)
   │  scripts/build-tiles.mjs
   │   - CSV·좌표 조인
   │   - 좌표 범위 검증(한국 영역), 중복 제거(1,683건)
   │   - 개방시간상세 우선 표시(요일 정보 포함 텍스트)
   │   - geohash 5자리(≈4.9km 격자) 단위로 분할
   ▼
public/tiles/{geohash5}.json × 4,356개 + _index.json
   + src/data-meta.json (생성 시각 기록, 앱 하단 "지도업데이트" 표시)
```

**갱신 주기**: 월 1회 권장(수동). 원본 CSV 재다운로드 → `npm run geocode` → `npm run tiles` → git commit/push → Vercel 자동 재배포.

## 6. 런타임 데이터 흐름

1. 앱 로드 시 `Geolocation API`로 현재 위치 획득(거부/실패 시 서울시청 폴백 + 안내 배너)
2. 현재 위치의 geohash 계산 → 인접 3×3 격자 타일 fetch → 결과 없으면 5×5로 자동 확장
3. Haversine 공식으로 거리 계산, 오름차순 정렬
4. 화면 폭에 따라 페이지 단위(모바일 3개/데스크톱 6개)로 하단 카드 표시
5. 지도는 **현재 카드 페이지에 해당하는 항목만** 기준으로 bounds 자동 조정(최소 반경 500m 보장, 과도한 축소는 상한선으로 방지)
6. 지도 마커는 카카오맵 Projection API로 화면 픽셀 좌표를 계산해, 근접한 마커는 골든 앵글 방식으로 밀어내 서로 가리지 않게 배치
7. 마커/카드 탭 → 상세 바텀시트(개방시간, 남녀 변기수, 장애인용, 기저귀교환대, 비상벨, CCTV, 관리기관 전화)
8. 개방시간 텍스트를 정규식 기반으로 파싱해 현재 이용 가능 여부 판정(모호하면 표시 생략 — 오탐 방지 우선)
9. "안내 시작" → 카카오내비 딥링크 시도(visibilitychange로 앱 전환 감지) → 실패 시 티맵 → 최종 카카오맵 웹 폴백(실제 GPS 있으면 출발지도 자동 채움)

## 7. 배포 아키텍처

- **GitHub → Vercel 연동**: `main` 브랜치에 push되면 Vercel이 자동으로 빌드(`npm run build`) 후 재배포
- **환경변수**: `VITE_KAKAO_JS_KEY`만 Vercel 프로젝트 Environment Variables에 등록(빌드 타임에 번들에 포함). 지오코딩용 키(`KAKAO_REST_KEY`, `VWORLD_KEY`)는 로컬 스크립트 전용이라 배포 환경에 불필요
- **카카오 도메인 화이트리스트**: 카카오 개발자센터 "플랫폼 키 → JavaScript 키 → JavaScript SDK 도메인"에 배포 도메인(`https://wc-sos.vercel.app`) 및 개발용(`http://localhost:5173`) 등록 필수 — 미등록 시 401(domain mismatched)로 지도 로드 실패
- **도메인**: Vercel 프로젝트명(`wcsos`)에 연결된 `wc-sos.vercel.app`을 기본 사용, 이전 프로젝트명 도메인(`restroommap.vercel.app`)도 계속 유효

## 8. 보안·프라이버시

- 위치정보는 기기 밖(자체 서버)으로 전송하지 않음 — 카카오맵 SDK·Places API 호출에 한해 위치가 쓰이며 이는 카카오 정책에 따름
- 관리자 인증·비밀번호 저장 기능 없음(버전1 범위 밖)
- 카카오 로컬 API로 변환한 좌표를 제3자에게 재공급(라이선싱)할 경우 이용약관 검토 필요(4장 참고, 현재는 카카오맵 표시 용도로만 사용해 문제 없음)

## 9. 확장성 설계(버전2~5 대비)

| 설계 요소 | 내용 |
|---|---|
| 데이터 스키마 예약 필드 | `source: "public"\|"user"`(버전2 제보), `category: "public"\|"subway"\|"private"`(버전3 지하철) |
| i18n 구조 | 모든 문자열이 `src/i18n/ko.json` 단일 파일로 관리 — 버전4 다국어 확장 시 언어별 파일 추가만으로 대응 |
| 지도 레이어 추상화 | `MapView.tsx`가 카카오맵 SDK 호출을 캡슐화 — 버전4~5 해외 확장 시 OSM/Leaflet 등으로 교체 가능한 구조 |
| 데이터 소스 추상화 | `src/lib/tiles.ts`가 정적 JSON fetch를 캡슐화 — 추후 서버 API로 교체 시 이 파일만 수정 |

상세 버전별 계획은 `06_버전2~5개발계획서.md` 참고.
