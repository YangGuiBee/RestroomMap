# 화장실SOS(RestroomMap) 향후 계획

| 항목 | 내용 |
|---|---|
| 문서버전 | v1.0 |
| 작성일 | 2026-07-17 |

버전2~5 기능 로드맵(`06_버전2-5_개발계획서.md`)과 별개로, 앱을 실제 서비스/사업으로 키워나갈 때 검토할 항목들을 정리한다.

---

## 1. APK(네이티브 앱)화

### 1.1 현재 상태
PWA(홈 화면 설치 가능)로 이미 앱과 유사하게 동작한다. 별도 스토어 배포 없이도 사용에는 지장 없다.

### 1.2 APK가 필요해지는 시점
- 구글플레이 노출을 통한 발견성(discoverability) 확보가 필요할 때
- 안드로이드오토/카플레이 연동을 시도할 때(아래 1.3 참고)
- 푸시 알림 등 PWA로 구현이 까다로운 네이티브 기능이 필요할 때

### 1.3 안드로이드오토/카플레이 — 별도 검토 필요
차량용 내비게이션 통합은 조사 결과 **별도 프로젝트 수준의 작업**이다.
- 일반 웹앱은 등록 자체가 불가능하고, Google Car App Library / CarPlay 템플릿을 쓰는 **네이티브 앱**만 허용
- "공중화장실 찾기"는 두 플랫폼이 정한 승인 카테고리(내비게이션/음악/메시징/주차/EV충전 등)에 명확히 속하지 않아 **심사 승인 자체가 불확실**
- 현재 차량 내장 브라우저(Tesla, 일부 현대·기아 인포테인먼트 등)로 `wc-sos.vercel.app` 접속은 이미 가능 — 이 경로가 현실적 대안

**권장**: 당장은 TWA(Trusted Web Activity)로 PWA를 감싸 플레이스토어에만 최소 등록하고, 안드로이드오토 대응은 실사용자 수요가 확인된 뒤 재검토.

## 2. 광고 추가

### 2.1 고려사항
- 긴급 상황용 앱 특성상, **광고로 인한 조작 지연·오터치는 안전 문제로 직결**된다. "실행→안내시작 터치 2회 이내"라는 핵심 UX 원칙과 정면으로 충돌할 소지가 큼
- 넣는다면 전면광고·배너보다는, 하단 카드 사이에 자연스럽게 섞이는 네이티브 광고(예: "이 근처 주유소" 스폰서 카드) 정도가 안전과의 절충점

### 2.2 권장 방향
버전1 단계에서는 광고 도입을 보류하고, 사용자 수·재방문율 지표가 쌓인 뒤(버전2 관리자 대시보드의 "사용 현황" 데이터 활용) 광고 도입 여부와 형태를 결정하는 것을 권장.

## 3. 지식재산권

### 3.1 상표 출원 — 권장
- 대상: 서비스명 "화장실SOS", 로고(restroom4.png 기반 아이콘)
- 이유: 이름·아이콘이 실제로 정해졌고 배포까지 완료된 상태라 출원 실익이 있음
- 참고: 특허청 특허고객상담센터(1544-8080) 무료 상담 활용 가능

### 3.2 저작권 등록 — 권장
- 대상: 소스코드(컴퓨터프로그램저작물), UI 디자인
- 창작과 동시에 자동 보호되지만, 한국저작권위원회(cros.or.kr) 등록 시 분쟁 발생 시 창작 시점 증거로 유용
- 현재 앱 하단에 저작권 표기(© 2026 guibee1004@gmail.com)를 이미 반영함

### 3.3 디자인 등록 — 선택
- 대상: 앱 아이콘, 고유 UI 디자인(있다면)
- 우선순위는 상표·저작권보다 낮음. 디자인이 충분히 독창적이라 판단되면 검토

### 3.4 특허 출원 — 비권장 (현재로서는)
- 조사 결과, 핵심 기능(위치기반 화장실 검색 + 내비 연동)은 이미 여러 서비스(화장실24, 똥마려, 대똥여지도, 생활안전지도 등)에 존재해 **신규성·진보성 확보가 어려움**
- 딥링크 폴백 체인(`kakaonavi://` → `tmap://` → 웹) 역시 이미 여러 앱이 쓰는 흔한 패턴이라 특허성 낮음
- 향후 정말 새로운 기술 요소(예: 독자적인 오프라인 데이터 압축·배포 방식 등)가 생기면 변리사 선행기술조사 후 재검토

## 4. 홍보 계획

### 4.1 채널
- 대학(건국대 공학대학원) 커뮤니티, 관련 수업/세미나를 통한 1차 배포
- 지역 커뮤니티(맘카페, 운전자 커뮤니티 등) — 실사용 후기 확보 목적
- SNS(카카오톡 채널, 인스타그램 등) 짧은 사용 시연 영상

### 4.2 차별점 메시지
경쟁 서비스 대비 강조할 지점(현대기아 제안서 작성 시 정리한 내용과 동일):
1. **긴급성 설계**: "실행 → 확인 → 안내 시작" 터치 2회 이내
2. **데이터 신뢰성**: 공공데이터 좌표 결측 문제를 자체 지오코딩으로 직접 해결(52,274건, 97.6%)
3. **정직한 정보 표시**: 불확실한 데이터는 숨기지 않고 원문 노출, 애매하면 "이용불가" 오탐 대신 표시 자체를 생략

### 4.3 B2B 제안 (참고)
현대자동차 H-Ignite, 기아 스타트업 협력 프로그램 등 오픈이노베이션 채널을 통한 데이터/기술 협력 제안서를 별도로 작성해둔 상태(`제안서_현대기아_오픈이노베이션.md`, 로컬 전용·비공개). 실제 제출 전 카카오 API 이용약관(좌표 재공급 제약) 법무 검토가 선결 과제.
