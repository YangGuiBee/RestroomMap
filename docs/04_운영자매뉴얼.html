1. 로컬 개발 환경

1.1 필요 항목

항목설명
Node.js18 이상 권장
.env 파일프로젝트 루트에 아래 3개 키 필요(.env.example 참고)
KAKAO_REST_KEY=...     # 지오코딩용 (카카오 개발자센터 > 플랫폼 키 > REST API 키)
VWORLD_KEY=...          # 지오코딩 폴백용 (VWorld Open API, 만료 2027-01-17)
VITE_KAKAO_JS_KEY=...   # 지도 표시용 (카카오 개발자센터 > 플랫폼 키 > JavaScript 키)

1.2 실행 명령어

cd C:\AI\RestroomMap
npm install         # 최초 1회
npm run dev          # 개발 서버, http://localhost:5173
npm run build         # 타입체크 + 프로덕션 빌드
npm run preview       # 빌드 결과 로컬 미리보기 (포트 4173)
localhost:5173은 카카오 개발자센터에 도메인 등록이 되어 있어 지도가 정상 표시됩니다. npm run preview(포트 4173)는 별도 등록이 안 되어 있어 지도가 401 에러로 안 뜰 수 있습니다 — 필요 시 4173도 등록하세요.

2. 배포 절차

GitHub main 브랜치에 push하면 Vercel이 자동으로 빌드·배포합니다. 별도 배포 명령이 필요 없습니다.

1코드 수정
2npm run build로 로컬에서 빌드/타입 오류 확인
3git add → git commit → git push origin main
4Vercel이 자동 재배포 (보통 30~90초 소요)
5https://wc-sos.vercel.app에서 강력 새로고침 후 반영 확인

2.1 환경변수 관리

Vercel 프로젝트 설정(Settings → Environment Variables)에 VITE_KAKAO_JS_KEY 1개만 등록되어 있으면 됩니다. KAKAO_REST_KEY, VWORLD_KEY는 로컬 지오코딩 스크립트 전용이라 배포 환경에는 불필요합니다.

3. 데이터 갱신 (권장 주기: 월 1회)

1공공데이터포털 "전국공중화장실표준데이터"에서 최신 CSV 재다운로드 → 공중화장실정보.csv로 교체
2CP949 → UTF-8 변환 후 data/공중화장실정보_utf8.csv로 저장
3npm run geocode 실행 — 신규/변경 주소만 지오코딩(resume 지원, 중단해도 재실행 시 이어서 처리)
4npm run tiles 실행 — public/tiles/ 전체 재생성 + src/data-meta.json 갱신일 자동 기록
5결과 리포트(제외 건수 등) 확인 후 git add → commit → push

4. 카카오 개발자센터 설정

4.1 새 배포 도메인을 추가할 때

Vercel에 새 도메인을 연결했다면(또는 프로젝트를 새로 만들었다면), 반드시 아래를 등록해야 지도가 뜹니다.

1Kakao Developers → 내 애플리케이션 → restroommap → 앱 설정 → 플랫폼 키
2Default JS Key 클릭 → JavaScript SDK 도메인에 새 URL 추가 (예: https://새도메인.vercel.app)
⚠️ 등록하지 않으면 401 domain mismatched 에러로 지도가 아예 안 뜹니다. 배포 직후 지도가 안 보이면 가장 먼저 이걸 의심하세요.

5. 트러블슈팅 (실제 겪었던 이슈)

증상원인해결
지도가 "네트워크를 확인하세요"로 안 뜸카카오 JS SDK 도메인 미등록4장 참고. curl로 Referer 헤더 넣어 401 여부로 진단 가능
지도 위 특정 번호 마커가 안 보임①동일/근접 좌표 마커가 서로 가림 ②고정 줌 레벨 때문에 뷰포트 밖으로 밀려남MapView.tsx에서 Projection API 기반 픽셀 충돌 회피(최소 간격 50px) + bounds 자동 맞춤으로 해결됨. 재발 시 MIN_PIXEL_GAP 상수를 더 키워볼 것
Vercel 재배포 후에도 옛날 화면이 보임배포 반영 지연(보통 30~90초) 또는 브라우저 캐시강력 새로고침(Ctrl+Shift+R), 배포 완료는 Vercel 대시보드 Deployments 탭에서 확인
새 Vercel 프로젝트명/도메인이 이미 사용 중*.vercel.app은 전역 네임스페이스라 타인이 선점 가능다른 이름으로 재시도 (대소문자는 무관 — 도메인은 대소문자 구분 안 함)

6. 롤백

git log --oneline          # 되돌릴 커밋 확인
git revert <커밋해시>        # 새 커밋으로 되돌리기 (권장 — 히스토리 보존)
git push origin main        # Vercel 자동 재배포

또는 Vercel 대시보드 Deployments 탭에서 과거 배포를 선택해 "Promote to Production"으로 즉시 되돌릴 수 있습니다(코드 되돌리기보다 빠름).
