## 1. 프로젝트 요약
- 목표: 운전 중 급할 때 가장 가까운 공공화장실을 3초 안에 찾는 **모바일 웹앱(PWA)**
- 스택: Vite + React + TS + vite-plugin-pwa, 카카오맵 JS SDK, 정적 타일 JSON(백엔드 없음)
- 상세계획: `화장실맵_버전1_개발계획.md` (Phase 0~6, 기술스택). 개발 규칙: `CLAUDE.md`
- 버전 로드맵: V1 공공화장실 → V2 민간 제보 → V3 지하철 → V4 다국어 → V5 해외
- 원칙: 지도 레이어 추상화(카카오↔OSM 교체 가능), 데이터 소스도 추상화(`lib/tiles.ts` → 추후 서버 API), i18n(`src/i18n/ko.json`), 위치정보 기기 밖 전송 안 함

## 3. 중요 발견/이슈
1. **공공데이터 좌표 제공 중단**(data.go.kr/15012892, 2025-02~) → 지오코딩 필수. 카카오 REST(일 10만) 사용해 완료.
2. **원본 품질**: 도로명/지번 뒤바뀜, 주소 꼬리말("평동 210 서대문역") 존재 → `geocode.mjs`가 교차·꼬리말 제거로 대응.
3. **지오코딩 실패 1,269건**: 지역 편중·주소결측 없음. `전남광주통합특별시` 등 신규 행정명 카카오 미매칭 다수 → 추후 주소 정규화로 일부 구제 가능(버전1.x).
4. **카카오 약관**: 변환 좌표를 카카오맵 외 지도에 영구 사용은 제약 소지. V1은 카카오맵 표시라 무관. 타 지도 전환 시(V4~5) VWorld 운영키 재지오코딩 권장.
5. **카카오맵 쿼터 정책 2026-07-21 변경** — 비즈월렛 연결 요구 여부 확인 필요. 초과 대비 지도 레이어 추상화로 Leaflet+OSM 전환 여지 확보.

## 6. 실행 명령어
```
cd C:\AI\RestroomMap
npm install                # 의존성 (최초/변경 시)
npm run dev                # 개발 서버 (localhost, 위치권한 허용됨)
npm run build              # 타입체크 + 프로덕션 빌드
npm run preview            # 빌드 결과 미리보기
npm run geocode            # 주소→위경도 (resume 지원, 재실행 시 이어짐)
npm run tiles              # geocoded.jsonl + CSV → public/tiles 재생성
```

## 7. 환경변수 (.env — 커밋 금지)
```
KAKAO_REST_KEY=...         # 지오코딩용 (설정됨)
VWORLD_KEY=...             # 지오코딩 폴백 (설정됨, 만료 2027-01-17)
VITE_KAKAO_JS_KEY=...      # 앱 지도 표시용 (플랫폼 키 > JavaScript 키). 설정 완료 + localhost:5173/배포도메인 등록 완료
```
> Vercel 배포용 환경변수는 로컬 `.env`와 별개로 **Vercel 대시보드 Project → Environment Variables**에도 동일하게 등록되어 있어야 함(`VITE_KAKAO_JS_KEY`만 필요, `KAKAO_REST_KEY`/`VWORLD_KEY`는 로컬 스크립트 전용이라 불필요). 이미 등록·배포 확인됨.
